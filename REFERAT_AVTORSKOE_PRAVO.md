### 2. ФУНКЦИОНАЛЬНОЕ ОПИСАНИЕ

#### 2.1. Модуль МСФО 9 - Финансовые инструменты

Программа реализует полный цикл расчета ожидаемых кредитных убытков (Expected Credit Loss, ECL) в соответствии с Международным стандартом финансовой отчетности 9 и Постановлением АРФР №269 от 29.12.2017.

**Ключевые функции:**
- Автоматическая 3-стадийная классификация финансовых активов
- Определение существенного увеличения кредитного риска (SICR)
- Расчет вероятности дефолта (PD) с макроэкономическими корректировками
- Моделирование потерь при дефолте (LGD) с учетом обеспечения
- Прогнозные сценарии (базовый, неблагоприятный, стрессовый)
- Взвешивание сценариев по вероятностям

**Математическая модель:**
```
ECL = Σ(t=1 to T) [PD_t × LGD × EAD_t × DF_t]

где:
PD_t  - вероятность дефолта в момент времени t
LGD   - потери при дефолте (69% необеспеченные, 45% обеспеченные)
EAD_t - подверженность дефолту в момент времени t
DF_t  - дисконтный фактор = exp(-r × t)
```

#### 2.2. Модуль МСФО 17 - Договоры страхования

Реализация трех подходов к оценке договоров страхования согласно МСФО 17:

1. **Общая модель оценки (GMM - General Measurement Model)**
   - Расчет наилучшей оценки обязательств (BEL)
   - Корректировка риска (RA) методами Cost of Capital и CTE
   - Контрактная сервисная маржа (CSM)
   - Определение обременительных договоров

2. **Подход переменной платы (VFA - Variable Fee Approach)**
   - Для участвующих договоров
   - Отслеживание справедливой стоимости базовых активов

3. **Подход распределения премии (PAA - Premium Allocation Approach)**
   - Для краткосрочных договоров (срок покрытия ≤ 1 год)
   - Упрощенный метод признания премий

**Ключевые формулы:**
```
Обязательство = FCF + CSM
FCF = BEL + RA
CSM = Премии - Затраты_на_привлечение - BEL - RA

Если CSM < 0: Договор обременительный → убыток признается немедленно
```

#### 2.3. Модуль Solvency 2 - Платежеспособность

Расчет показателей достаточности капитала в соответствии с Постановлением АРФР №304 от 26.12.2016 и требованиями EIOPA.

**Основные показатели:**
- ММП (минимальная маржа платежеспособности)
- ФМП (фактическая маржа платежеспособности)
- Коэффициент Nмп = ФМП / ММП (норматив ≥ 1.0)

**Расчет ММП:**
```
ММП_P = Премии × k × 0.18        (на основе премий)
ММП_I = Понесенные_убытки × k × 0.26 (на основе убытков)

ММП = max(ММП_P, ММП_I) + ОСАГО_корректировка

где k - коэффициент удержания (обычно 0.70)
```

**Стресс-тестирование:**
- Adverse сценарий (неблагоприятный)
- Severe сценарий (серьезный)
- VaR 99.5% (Value at Risk с доверительным интервалом 99.5%)

#### 2.4. Модуль ФГСВ

Калькулятор взносов в Фонд гарантирования страховых выплат согласно Закону РК №423-II от 03.06.2003.

**Функции:**
- Расчет базового взноса на основе собранных премий
- Применение мультипликаторов в зависимости от платежеспособности
- Штрафные мультипликаторы при нарушениях
- Моделирование банкротства страховщиков
- Симуляция выплат страхователям

### 4. АЛГОРИТМИЧЕСКАЯ НОВИЗНА

#### 4.1. Адаптация международных стандартов к казахстанской специфике

Программа уникальна тем, что адаптирует международные стандарты МСФО 9, МСФО 17 и Solvency 2 к специфике казахстанского страхового рынка:

1. **Интеграция макроэкономических показателей НБК**
   - Базовая ставка НБК (18.0% на декабрь 2025)
   - Инфляция (12.9%)
   - Курс USD/KZT (560.00)
   - Цена нефти Brent

2. **Учет специфики ОСАГО в расчетах ММП**
   - Корректировка минимальной маржи для компаний с ОСАГО
   - Особые требования к резервированию

3. **Взаимосвязь МСФО и Solvency 2**
   - Автоматическая корректировка ФМП на величину ECL (МСФО 9)
   - Учет CSM (МСФО 17) в собственных средствах

#### 4.2. Многосценарный анализ с взвешиванием

Уникальная методология прогнозного расчета ECL:

```
PD_adjusted = PD_base × [w1×M_baseline + w2×M_adverse + w3×M_severe]

где:
w1 = 55%, M_baseline = 1.35 (базовый сценарий)
w2 = 35%, M_adverse  = 1.80 (неблагоприятный)
w3 = 10%, M_severe   = 2.40 (стрессовый)

Итоговый взвешенный мультипликатор = 1.613
```

Такой подход обеспечивает более точную оценку кредитных рисков по сравнению с использованием единственного сценария.

#### 4.3. Автоматическая классификация SICR

Программа автоматически определяет существенное увеличение кредитного риска (SICR) на основе двух критериев:

1. **Количественный критерий**: увеличение PD более чем в 2 раза (>100%)
2. **Качественный критерий**: просрочка платежа 30-89 дней

При выполнении любого из критериев актив переводится из Стадии 1 в Стадию 2 с переходом от 12-месячного ECL к пожизненному.

### 7. ПРАКТИЧЕСКОЕ ПРИМЕНЕНИЕ

#### 7.1. Сценарии использования

**Сценарий 1: Ежеквартальная отчетность в АРФР**

Страховая компания использует программу для:
1. Расчета ECL по портфелю финансовых активов (облигации, депозиты, займы)
2. Оценки обязательств по договорам страхования (BEL + RA + CSM)
3. Расчета коэффициента платежеспособности Nмп
4. Генерации отчетов в формате XBRL для подачи в АРФР

**Сценарий 2: Стресс-тестирование**

АРФР проводит стресс-тест всего рынка:
1. Задает параметры стрессового сценария (рост PD на 50%, падение рынка на 30%)
2. Программа автоматически пересчитывает показатели всех страховщиков
3. Выявляются компании с Nмп < 1.0 в стрессовых условиях
4. Формируются рекомендации по докапитализации

**Сценарий 3: Расчет взносов в ФГСВ**

ФГСВ использует программу для:
1. Получения данных о платежеспособности всех страховщиков
2. Расчета базовых взносов на основе собранных премий
3. Применения повышающих коэффициентов для компаний с низким Nмп
4. Моделирования достаточности фонда при банкротстве 1-2 страховщиков

#### 7.2. Экономический эффект

Использование программы обеспечивает:

1. **Снижение затрат на соответствие** - автоматизация расчетов экономит 60-80% времени актуариев
2. **Сокращение ошибок** - устранение ручных расчетов снижает риск ошибок на 95%
3. **Ускорение отчетности** - подготовка квартальной отчетности за 1 день вместо 5-7 дней
4. **Улучшение управления рисками** - ежедневный мониторинг вместо ежеквартального
